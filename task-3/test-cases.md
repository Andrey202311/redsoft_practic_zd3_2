# Задание 3: Тест-кейсы «Восстановление пароля»

---

## Часть 1. Уточняющие вопросы к аналитику/заказчику

| № | Вопрос | Предполагаемый ответ |
|---|--------|----------------------|
| 1 | Можно ли использовать повторно ссылку после успешной смены пароля? | Нет, ссылка становится недействительной (FR-9) |
| 2 | Сколько попыток ввода невалидного пароля доступно на странице установки? | Не ограничено, пока не закроется страница |
| 3 | Что происходит при повторном запросе сброса: старая ссылка отзывается или остаётся активной? | Старая ссылка отзывается, отправляется новая |
| 4 | Есть ли ограничение на частоту запросов сброса пароля (rate limiting)? | Да, не чаще 1 раза в 5 минут с одного IP |
| 5 | Отображается ли email получателя в письме для сброса пароля? | Да, для подтверждения адресата |
| 6 | Что происходит при переходе по уже использованной ссылке? | Сообщение «Ссылка уже использована» |
| 7 | Можно ли открыть страницу установки пароля без перехода по ссылке из письма? | Нет, страница доступна только по уникальной ссылке |

## Часть 2. Тест-кейсы

---

### Позитивные тест-кейсы

#### TC-01: Успешный запрос на сброс пароля

**Цель тестирования**
Проверка отправки письма со ссылкой для сброса пароля на зарегистрированный email.

**Предварительные условия**
1. Пользователь не авторизован в системе.
2. В системе существует пользователь с email user@example.com.

**Процедура тестирования**
1. Перейти на страницу входа (FR-1).
2. Нажать ссылку «Забыли пароль?» (FR-2).
3. В поле email ввести user@example.com.
4. Нажать кнопку «Отправить ссылку» (FR-3).

**Ожидаемый результат**
1. Отображается универсальное сообщение: «Если указанный email зарегистрирован в системе, на него было отправлено письмо с инструкциями» (FR-3).
2. На email приходит письмо с уникальной ссылкой для сброса пароля.

**Покрытие требований:** FR-1, FR-2, FR-3

---

#### TC-02: Успешная установка нового пароля по ссылке

**Цель тестирования**
Проверка успешной смены пароля с валидными данными.

**Предварительные условия**
1. Пользователь получил письмо со ссылкой.
2. Ссылка не просрочена (менее 1 часа, FR-4).
3. Ссылка не использовалась ранее.

**Процедура тестирования**
1. Перейти по ссылке из письма (FR-5).
2. В поле «Новый пароль» ввести «NewPass1» (FR-7: ≥8 символов, заглавная, цифра).
3. В поле «Подтвердить новый пароль» ввести «NewPass1» (FR-6).
4. Нажать «Сохранить новый пароль» (FR-8).

**Ожидаемый результат**
1. Пароль пользователя изменяется.
2. Отображается сообщение об успешном изменении пароля (FR-8).
3. Автоматический редирект на страницу входа (FR-8).
4. Пользователь может войти с новым паролем.

**Покрытие требований:** FR-4, FR-5, FR-6, FR-7, FR-8

---

#### TC-03: Повторный запрос сброса отзывает старую ссылку

**Цель тестирования**
Проверка, что повторный запрос сброса делает старую ссылку недействительной.

**Предварительные условия**
1. Пользователь уже запросил сброс пароля.
2. Первое письмо получено, ссылка не использована.

**Процедура тестирования**
1. Нажать «Забыли пароль?» повторно.
2. Ввести тот же email.
3. Нажать «Отправить ссылку».
4. Перейти по ссылке из первого письма.

**Ожидаемый результат**
1. Отправлено новое письмо.
2. Переход по старой ссылке показывает сообщение «Ссылка устарела».

**Покрытие требований:** FR-3, FR-9

---

### Негативные тест-кейсы

#### TC-04: Незарегистрированный email

**Цель тестирования**
Проверка обработки запроса сброса для незарегистрированного email.

**Предварительные условия**
Пользователь не авторизован.

**Процедура тестирования**
1. Нажать «Забыли пароль?».
2. Ввести email unknown@example.com.
3. Нажать «Отправить ссылку».

**Ожидаемый результат**
1. Отображается универсальное сообщение (FR-3).
2. Письмо не отправляется на незарегистрированный email.

**Покрытие требований:** FR-3

---

#### TC-05: Невалидный формат email

**Цель тестирования**
Проверка валидации формата email при запросе сброса.

**Предварительные условия**
Пользователь не авторизован.

**Процедура тестирования**
1. Нажать «Забыли пароль?».
2. Ввести «invalid-email» (без @).
3. Нажать «Отправить ссылку».

**Ожидаемый результат**
1. Отображается сообщение: «Введите корректный email».
2. Кнопка «Отправить» неактивна или запрос не отправляется.

**Покрытие требований:** FR-2, FR-3

---

#### TC-06: Пароль короче 8 символов

**Цель тестирования**
Проверка валидации минимальной длины пароля.

**Предварительные условия**
Пользователь перешёл по валидной ссылке (FR-5).

**Процедура тестирования**
1. Ввести «Pass1» в оба поля пароля.
2. Нажать «Сохранить».

**Ожидаемый результат**
1. Отображается ошибка: «Пароль должен содержать минимум 8 символов» (FR-7).
2. Пароль не изменяется.

**Покрытие требований:** FR-7

---

#### TC-07: Пароль без заглавной буквы

**Цель тестирования**
Проверка требования наличия заглавной буквы в пароле.

**Предварительные условия**

**Процедура тестирования**

2. Нажать «Сохранить».

**Ожидаемый результат**
1. Ошибка: «Пароль должен содержать минимум одну заглавную букву» (FR-7).
2. Пароль не изменяется.

**Покрытие требований:** FR-7

---

**Цель тестирования**
Проверка требования наличия цифры в пароле.

**Предварительные условия**
Пользователь перешёл по валидной ссылке.

**Процедура тестирования**
1. Ввести «NewPassword» в оба поля.
2. Нажать «Сохранить».

**Ожидаемый результат**

---

#### TC-09: Несовпадающие пароли

**Цель тестирования**
Проверка валидации совпадения полей пароля.

**Предварительные условия**
Пользователь перешёл по валидной ссылке.

**Процедура тестирования**
1. Ввести «NewPass1» в «Новый пароль».
2. Ввести «Different2» в «Подтвердить».
3. Нажать «Сохранить».

**Ожидаемый результат**
1. Ошибка: «Пароли не совпадают» (FR-8).
2. Пароль не изменяется.

**Покрытие требований:** FR-6, FR-8

### Тест-кейсы на состояние системы

#### TC-10: Просроченная ссылка (граничное условие)

**Цель тестирования**
Проверка поведения системы при переходе по ссылке после истечения срока действия (1 час).

**Предварительные условия**
1. Пользователь получил письмо.
2. Прошло более 1 часа с момента отправки (FR-4).

**Процедура тестирования**
1. Открыть письмо.
2. Перейти по ссылке.

**Ожидаемый результат**
1. Отображается сообщение: «Ссылка устарела. Запросите сброс пароля повторно».
2. Форма установки пароля не отображается.

**Покрытие требований:** FR-4

---

#### TC-11: Использованная ссылка (состояние системы)

**Цель тестирования**
Проверка поведения системы при повторном переходе по уже использованной ссылке.

**Предварительные условия**
1. Пользователь успешно сменил пароль по ссылке (FR-9).
2. Ссылка уже использована.

**Процедура тестирования**
1. Повторно перейти по той же ссылке из письма.

**Ожидаемый результат**
1. Отображается сообщение: «Ссылка уже использована» или «Ссылка устарела».
2. Форма установки пароля не отображается.

**Покрытие требований:** FR-9

## Часть 3. Критичные сценарии для смоук-тестов

| № | Сценарий | Приоритет | Обоснование |
|---|----------|-----------|-------------|
| 1 | Успешный запрос сброса + получение письма (TC-01) | **Критичный** | Базовая функция восстановления доступа |
